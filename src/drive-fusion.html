Browse, search, and move files across several Drive accounts in one workspace.

Drive Fusion is a web UI concept for users who operate more than one Google account and want a single operational layer for indexing files, viewing quotas, searching content, and planning transfers without switching between accounts repeatedly.

Project overview

This project does not literally merge storage quotas into one Google account. Instead, it builds a federated workspace on top of multiple separately authorized Drive accounts, so users can search, inspect, upload, download, and transfer content through one interface while each account keeps its own Google-managed quota and ownership boundary.

Federation

Unified index

Normalize files from each connected account into one metadata catalog for fast cross-account search, filtering, and sorting.

Operations

Transfer planner

Stage copy and move jobs between accounts with clear source and destination ownership, size forecasts, and progress visibility.

Governance

Quota intelligence

Track used, free, and projected storage per connected account so users know where new uploads should land.

Core features

The interface focuses on operational convenience rather than pretending multiple Drive accounts become one physical volume.

Workflow

The product experience is designed around five operational steps.

1

Connect accounts

Users sign in with one or more Google accounts through OAuth and select the Drive scopes required for metadata, file actions, and uploads.

2

Index workspace

The system fetches file metadata, folder trees, labels, and quota usage for each connected Drive and writes a normalized index.

3

Search and organize

Users filter by owner account, mime type, modified date, folder path, and storage pressure to decide how to rebalance data.

4

Run transfer jobs

Selected files are copied between accounts with status tracking, byte-level progress, and conflict handling.

5

Export reports

The workspace can export storage summaries, transfer history, and file inventories for recordkeeping and review.

6

Disconnect safely

Tokens are revoked, local metadata is deleted on request, and the workspace returns to an unlinked state.
